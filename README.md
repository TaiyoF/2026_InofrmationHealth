## UI design rules

- The theme picker page surface follows the currently selected theme, while every option card is always rendered in its own theme: Basic stays neutral, Media Quest stays navy/gold, and Wanko Challenge stays light blue/cream. Selecting one card must update the surrounding settings surface without recoloring the other option cards.
- Mobile must expose Theme as a labeled item in the Home bottom navigation and as a labeled action in Profile. Do not rely on an unlabeled gear icon or a Profile-only route for theme discovery.
- Each theme must keep readable foreground/background contrast across home, check, reaction, profile, and settings screens.
- Reaction choices use the shared line-icon system instead of emoji standing in for interface icons.
- Badge artwork is theme-specific: provided glossy rank images belong to Basic; Media Quest uses its own game emblems. The badge artwork is rendered as a frame outside the avatar, with the person icon or account mark layered above it throughout the feed. Adding a frame must never shrink, move, or resize the avatar circle: the opaque icon core keeps exactly the same width, height, and position as an unframed avatar, while only the frame pseudo-element may be aligned or scaled. The icon layer must not allow the badge artwork to show through its center. Media Quest uses one reusable hollow hexagonal frame asset and the same composition for both Profile badges and Home avatar frames: a full-size opaque circular core above the frame, with only the outer ring visible. Profile rank badges must not expose inner frame rings that the avatar version hides. Rank colors may change, but the frame silhouette, crop, scale, and shadow may not. Its avatar core always uses a meaningful line icon instead of a one-character initial. In the feed, the visible Media Quest frame should extend only about 5 px beyond the 38 px icon core so it does not crowd the account name or post content. Wanko Challenge badges use the user-provided Silver badge crop as the canonical visual source instead of regenerating or redrawing its geometry. Bronze, Silver, Gold, Platinum, and Black are five separate rank-colored raster assets derived from that same crop; changing rank must change the ring and face color while preserving the exact circle, star, and compact coral-ribbon silhouette. Current-season medals, badge-track medals, profile avatars, feed avatars, and result-page avatars all reuse that silhouette and proportional crop. Avatar versions place the unchanged full-size opaque account icon over the matching rank asset so the icon never shrinks and the source star does not show through. The Wanko avatar-frame placement is calculated from the circle in the 288 x 272 source asset, whose center is approximately (151.5, 112), rather than from the asymmetric full image bounds that include ribbons. For the standard icon core, the compact frame uses width 160%, height 151.11%, left -34.17%, and top -12.22%; this keeps the medal center fixed, limits the visible ring to about 3.8 px beyond a 38 px icon core, and reduces the ribbon footprint by about 6%. Change only these frame values when alignment or frame size needs correction. Before publishing a Wanko badge change, compare the rendered Silver current badge, the five-color badge track, and at least one feed avatar against the supplied references at their actual display sizes; deployment and asset existence alone are not visual verification. Basic rank artwork is displayed slightly larger than the other themes without cropping or distorting the supplied circular images. Rank badges must preserve their source aspect ratio at every size and theme; do not stretch the circular badge face. Image-backed badges must set background position, size, and repeat explicitly and be verified at both current-rank and badge-track sizes. A rank shown in Profile and the same rank used as a Home avatar frame must use the identical artwork, crop, scale, silhouette, and shadow; only the opaque account icon overlay may differ.
- The same screen must keep the same component hierarchy and visual language on desktop and mobile. Responsive rules may change available width, padding, and navigation placement, but must not substitute different headers, cards, badges, typography, or controls for the same content.
- Post cards should start with the avatar/account row and must not add category chips, “today's item” labels, or other tag-like metadata above it. Keep category and progress context in the surrounding challenge UI instead of duplicating it inside the post.
- Post action bars must stay on one line at every supported width. Show only the action icon and count visually; keep names such as Like, Comment, Repost, Save, and Share in accessible labels and tooltips.
- Search, Notifications, and Messages are dormant demo chrome rather than working features. Their navigation controls stay visible but disabled and visually muted, and their desktop/mobile screens contain no sample records, inputs, or actions beyond a subdued unavailable state.
- Responsive desktop layouts must prioritize the central feed. As width decreases, hide the right profile panel first, then collapse the left navigation to an icon rail with an accessible open/close control, and only switch to the mobile shell at the smallest breakpoint.
- Media Quest must never place yellow or gold text on white or pale surfaces. Use dark navy text on light settings surfaces, and reserve gold text and accents for dark navy surfaces with sufficient contrast.
- Incomplete navigation states use a small theme-colored notification dot, not a text badge such as "未". Keep the full state in the control's accessible label.
- Checked-post cards contain only the account, post content, and reaction actions. Keep explanatory copy outside the card and leave clear space between the card and subsequent navigation buttons.
